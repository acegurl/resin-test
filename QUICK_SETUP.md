## 🚀 Setup (3 Steps)

### Step 1: Deploy Dashboard
Replace your current `index.html` with the updated version

```bash
cp index.html index.html  # from this package
```

### Step 2: Generate Data
Create your first `data.json` file using the script:

```bash
# Ensure plastic-resin-raw.xlsx is in the same directory
python3 gen_data.py

# This creates data.json with all 7 subcategory averages
```

### Step 3: Serve Dashboard
Upload `index.html` and `data.json` to your server/hosting

```bash
# The dashboard loads data.json automatically
# No additional configuration needed
```

Each day (30 seconds):

```bash
# 1. Edit your raw data
```

## 🔧 How It Works

```
plastic-resin-raw.xlsx (raw2 sheet)
    ↓
    [Products marked with "For Avg" subcategory]
    ↓
    gen_data.py calculates:
      Average = Sum(prices) / Count(prices)
      Delta = (Today Avg - Yesterday Avg)
    ↓
    data.json (7 subcategories × all dates)
    ↓
    index.html loads and displays
```

## 📋 Data Structure

The `data.json` file has this structure:

```json
{
  "labels": ["6/25/2026", "6/26/2026"],
  "summary": {
    "subcats": {
      "PP injection": {
        "avg": 32600,
        "delta": -900
      },
      "PP yarn": {
        "avg": 34700,
        "delta": -214
      },
      ...
    }
  }
}
```

The `gen_data.py` script generates this automatically from your Excel file.

## 🆘 Troubleshooting

**Problem: gen_data.py fails**
- Check: `plastic-resin-raw.xlsx` exists in same folder
- Check: Python 3.8+ with pandas installed
- Solution: `pip install pandas openpyxl`

**Problem: Dashboard shows "—" for all prices**
- Check: data.json is in same folder as index.html
- Check: Browser console for errors (F12)
- Solution: Reload page (Ctrl+Shift+R)

**Problem: Only 3 cards show instead of 7**
- Check: data.json has "summary.subcats" object
- Check: All 7 subcategory names are present
- Solution: Re-run gen_data.py

**Problem: Subcategory names show "PP", "PE", "PVC"**
- This means old data.json is still loaded
- Solution: Delete old data.json, run gen_data.py again
